# Portfolio screenshot checklist

This directory intentionally contains instructions instead of fabricated screenshots. Capture the running product with demo-only data after the final deployment or local release build.

Recommended filenames:

- `01-home.png`
- `02-dashboard.png`
- `03-resumes.png`
- `04-resume-builder.png`
- `05-applications.png`
- `06-resume-analysis.png`
- `07-best-resume.png`
- `08-interviews-reminders.png`
- `09-analytics.png`
- `10-job-post-review.png`

Use a 1440 × 900 desktop viewport for the main set and include one 390 × 844 mobile dashboard/application view if useful. Crop browser chrome consistently, keep each optimized image reasonably small, and verify that no screenshot contains a real name, email address, resume, phone number, employer note, access token, connection string, or local filesystem path.
